THEME FUNDAMENTALS

What Are Themes?
Themes allow your design system to adapt to different contexts:
• Light/Dark Mode - Support user preferences
• Brand Variations - Multi-brand products
• Accessibility - High contrast and accessible themes
• Context-Specific - Seasonal or event-based themes

Theme Architecture
Learn the three-tier approach to theming:
1. Primitive Tokens - Base values (colors, sizes, typography)
2. Semantic Tokens - Purpose-based tokens with mode variations
3. Component Tokens - Component-specific theme overrides

COMMON THEMING PATTERNS

Light/Dark Theme Example:
Primitive: gray-50 to gray-900
Semantic (Light): background = gray-50, text = gray-900
Semantic (Dark): background = gray-900, text = gray-50

Multi-Brand Theme Example:
Brand Collection (Brand A / Brand B):
• primary: blue-600 (A) / red-600 (B)
• secondary: blue-400 (A) / red-400 (B)

Accessibility Theme Example:
High Contrast Mode:
• Increased contrast ratios
• WCAG AAA compliant colors
• Enhanced focus indicators
